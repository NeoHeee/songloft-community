# Songloft 自定义主题包规范

Songloft 主题包使用声明式 JSON，不允许脚本、CSS、远程链接或可执行代码。建议文件扩展名使用 `.songloft-theme`，也可直接导入 `.json` 文件。

## 当前能力

- 同一份主题包可用于 Web、Android、Windows、Linux、macOS 和 iOS。
- 主题包同时提供浅色与深色配置，用户仍可独立选择“浅色 / 深色 / 跟随系统”。
- 导入前显示主题名称、作者、版本、主题 ID、浅深色预览和描述。
- 检测到相同自定义主题 ID 时，会明确提示“覆盖更新”，确认后才写入。
- 可导出当前主题或任意已安装主题为 `.songloft-theme` 文件。
- 可复制任意主题的标准 JSON，方便二次制作和在线发布。
- 主题包只负责视觉变量，不改变播放、插件、网络和数据逻辑。
- 导入前执行字段、颜色、圆角与大小校验。

## 限制

- 当前规范版本：`schemaVersion: 1`
- 单个主题包最大：128 KB
- 最多安装：32 个自定义主题包
- `id` 长度：3-64 个字符
- `id` 仅允许：小写字母、数字、点、下划线、短横线
- 颜色格式：`#RRGGBB` 或 `#AARRGGBB`
- 圆角范围：0-40
- `light` 与 `dark` 必须同时存在
- 自定义主题不能使用内置主题的 `id`

## 完整示例

```json
{
  "schemaVersion": 1,
  "id": "my-theme-pack",
  "name": "我的主题",
  "version": "1.0.0",
  "author": "作者名称",
  "description": "主题简介",
  "light": {
    "seed": "#7C5CFF",
    "background": "#F4F5FA",
    "surface": "#FFFFFF",
    "secondary": "#4C7DFF",
    "tertiary": "#B45CFF",
    "playerGradient": ["#7C5CFF", "#4C7DFF"],
    "cardRadius": 22,
    "controlRadius": 15,
    "navigationRadius": 16
  },
  "dark": {
    "seed": "#9C87FF",
    "background": "#0B0D12",
    "surface": "#151821",
    "secondary": "#6E9BFF",
    "tertiary": "#D18AFF",
    "playerGradient": ["#8B6CFF", "#426FE8"],
    "cardRadius": 22,
    "controlRadius": 15,
    "navigationRadius": 16
  }
}
```

## 字段说明

### 顶层字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `schemaVersion` | 是 | 当前固定为 `1` |
| `id` | 是 | 主题唯一标识；同一自定义 `id` 再次导入会覆盖更新 |
| `name` | 是 | 主题显示名称，最多 48 个字符 |
| `version` | 是 | 主题版本号，最多 24 个字符 |
| `author` | 是 | 作者名称，最多 48 个字符 |
| `description` | 否 | 主题简介，最多 160 个字符 |
| `light` | 是 | 浅色主题配置 |
| `dark` | 是 | 深色主题配置 |

### 配色字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `seed` | 是 | Material 3 主种子色 |
| `background` | 是 | 页面背景色 |
| `surface` | 是 | 卡片、侧栏、弹窗等面板色 |
| `secondary` | 否 | 次强调色；省略时由 `seed` 自动生成 |
| `tertiary` | 否 | 第三强调色；省略时由 `seed` 自动生成 |
| `playerGradient` | 否 | 两个颜色组成的渐变；用于品牌标识和播放器视觉 |
| `cardRadius` | 否 | 卡片与弹窗圆角，默认 22 |
| `controlRadius` | 否 | 输入框与按钮圆角，默认 15 |
| `navigationRadius` | 否 | 底栏、侧栏选中块圆角，默认 16 |

## 制作步骤

1. 在应用中进入“设置 → 外观设置 → 主题包 → 制作规范”。
2. 点击“复制模板”，或复制仓库中的示例文件。
3. 修改 `id`、名称、作者、版本和配色。
4. 保存为 UTF-8 编码文本文件。
5. 将扩展名改为 `.songloft-theme`。
6. 在 Songloft 中点击“导入主题包”。
7. 检查导入预览中的作者、版本、主题 ID 和浅深色效果。
8. 点击“安装并启用”；相同自定义 ID 会显示“更新并启用”。

## 导出与分享

### 导出当前主题

在“设置 → 外观设置 → 主题包”中点击“导出当前主题”，选择保存位置后会生成标准 `.songloft-theme` 文件。

### 导出指定主题

点击主题卡右上角菜单，选择“导出主题包”。内置主题和自定义主题都可以导出。

### 复制 JSON

点击主题卡右上角菜单，选择“复制 JSON”。复制的内容可以直接粘贴到文本编辑器中继续修改。

内置主题导出的 ID 属于系统保留 ID，不能作为自定义主题直接导入。基于内置主题二次制作时，必须先修改 `id`、`name`、`version` 和 `author`。

## 推荐发布规则

准备把主题上传到 GitHub、论坛或主题仓库时，建议同时提供：

- `.songloft-theme` 文件
- 主题名称和版本
- 作者或维护者名称
- 浅色与深色截图
- 支持的 `schemaVersion`
- 更新说明
- 许可证或允许转载说明

推荐文件名：

```text
<theme-id>-v<version>.songloft-theme
```

例如：

```text
community-aurora-purple-v1.0.0.songloft-theme
```

## 版本更新规则

- 修正颜色或圆角：增加补丁版本，例如 `1.0.0 → 1.0.1`
- 增加较多视觉调整：增加次版本，例如 `1.0.1 → 1.1.0`
- 主题风格完全重做或不兼容：增加主版本，例如 `1.1.0 → 2.0.0`
- 更新主题时保持原 `id`，Songloft 才能识别为覆盖更新
- 新主题或分支主题必须使用新的 `id`

## 安全规则

主题包不会解析或执行以下内容：

- JavaScript、Dart 或其他脚本
- CSS
- HTML
- 网络请求地址
- 本地文件路径
- 动态字体和可执行插件

主题包导入仍应来自可信来源。虽然当前格式不执行代码，但恶意或极端配色可能降低界面可读性；导入前应检查应用提供的浅深色预览和作者信息。

后续规范版本可增加打包图片资源，但必须继续采用白名单字段、文件大小限制、资源数量限制和路径校验。
