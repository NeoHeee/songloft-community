# Songloft UI Redesign

本分支用于在保留 Songloft 现有后端、播放、认证和插件逻辑的前提下，重构客户端视觉与交互。

## 第一阶段（已完成）

- 重建亮色与暗色主题：紫蓝强调色、沉浸式背景、统一圆角与卡片层级。
- 重做移动端迷你播放器：悬浮卡片、加大封面、下一首快捷操作、底部进度条。
- 重做桌面端播放器：悬浮播放坞、强化歌曲信息区和控制区、保留原工具栏能力。

## 第二阶段（已完成）

- 新增桌面/平板新版应用外壳。
- 桌面端采用品牌侧边栏 + 内容区 + 播放队列 + 底部播放坞布局。
- 平板端使用紧凑图标导航，保持更多内容空间。
- 重构首页欢迎区、音乐库状态、快捷入口、歌单网格和封面占位样式。
- 保留插件页面在 Web 端的 Offstage 保活机制。

## 第三阶段（已完成）

- 重构歌曲库类型筛选栏和歌曲数量显示。
- 重构移动端歌曲卡片、桌面歌曲行、类型标签与当前播放状态。
- 统一导航图标为新版圆角风格。
- 重构歌单网格卡片与列表视图，保留播放、多选、编辑、隐藏和删除能力。
- 重构设置页移动端入口、桌面分类侧栏和设置分组卡片。
- 重构插件快捷入口网格。

## 第四阶段（已完成）

- 重构歌曲库顶部搜索和操作面板。
- 修复搜索框清除按钮不能即时刷新的问题。
- 重构桌面/平板全屏播放器，采用封面信息卡、沉浸歌词和玻璃控制台布局。
- 重构歌词加载、空状态、失败状态和当前歌词焦点样式。
- 重构歌单详情页：宽屏采用歌单信息侧栏与歌曲工作区，移动端采用沉浸封面卡片。
- 重构歌单内搜索、播放、添加、多选、批量删除和永久排序操作。
- 重构插件管理：状态统计、搜索筛选、响应式卡片、上传、更新、强制安装和清理残留。

## 工程验证

- GitHub Actions 自动执行依赖安装、Dart 格式检查和 `flutter analyze`。
- 静态分析报告会作为工作流附件保存。
- Flutter Web Release 已完成编译验证，并上传 `songloft-web-preview`。
- Android Release 已完成三种 ABI 构建：`arm64-v8a`、`armeabi-v7a`、`x86_64`。
- Windows x64 Release 已完成编译并生成便携 ZIP。
- Web、Android、Windows 预览产物默认保留 14 天。
- Android 与 Windows 构建采用按需触发，避免普通 UI 提交重复消耗构建资源。

## 后续阶段

1. 完成手机、平板、桌面端视觉回归和交互细节修正。
2. 增加 Docker / Songloft 服务端内嵌 Web 构建验证。
3. 完成正式品牌名称、应用图标和启动页替换。
4. 整理合并说明，并评估是否合并到 `main`。

## 设计原则

- UI 与业务状态解耦，尽量复用现有 Riverpod Provider。
- 不改变现有 REST API 和插件协议。
- 桌面、平板、手机和 TV 保持响应式能力。
- 每个阶段独立提交，便于回滚和跟进上游更新。
